
### 1. Failed Logon Attempts (Potential Brute Force)

Identify accounts experiencing a high number of failed logon attempts.

```kql
SecurityEvent
| where EventID == 4625
| summarize FailedAttempts = count() by Account, IpAddress
| where FailedAttempts > 10
| sort by FailedAttempts desc
```

---

### 2. Successful Logons After Multiple Failures

Detect potential password-spraying or brute-force success.

```kql
let FailedLogons =
SecurityEvent
| where EventID == 4625
| summarize FailedCount = count() by Account
| where FailedCount > 10;

let SuccessfulLogons =
SecurityEvent
| where EventID == 4624
| project Account, TimeGenerated;

FailedLogons
| join kind=inner SuccessfulLogons on Account
```

---

### 3. User Logon and Logoff Correlation

Compare user login and logout activity.

```kql
SecurityEvent
| where EventID == 4624
| summarize LogonCount=count() by Account
| join kind=inner (
    SecurityEvent
    | where EventID == 4634
    | summarize LogoffCount=count() by Account
) on Account
| project Account, LogonCount, LogoffCount
```

---

### 4. New Administrative Account Creation

Detect creation of new privileged accounts.

```kql
SecurityEvent
| where EventID == 4720
| project TimeGenerated, Computer, Account, TargetAccount
```

---

### 5. Privilege Escalation Activities

Monitor users added to privileged groups.

```kql
SecurityEvent
| where EventID in (4728, 4732, 4756)
| project TimeGenerated, Account, TargetAccount, Computer
```

---

### 6. Account Lockout Monitoring

Identify accounts repeatedly being locked.

```kql
SecurityEvent
| where EventID == 4740
| summarize LockoutCount=count() by TargetAccount
| sort by LockoutCount desc
```

---

### 7. PowerShell Execution Detection

Detect PowerShell activity that may indicate attacker behavior.

```kql
SecurityEvent
| where Process has "powershell.exe"
| project TimeGenerated, Computer, Account, Process
```

---

### 8. Suspicious Encoded PowerShell Commands

Detect Base64-encoded PowerShell execution.

```kql
SecurityEvent
| where Process has "powershell.exe"
| where Process has_any ("-enc", "-encodedcommand")
| project TimeGenerated, Computer, Account, Process
```

---

### 9. Suspicious Command-Line Activity

Identify common attacker tools and commands.

```kql
SecurityEvent
| where Process has_any (
    "mimikatz",
    "psexec",
    "net user",
    "whoami",
    "nltest",
    "rundll32",
    "certutil"
)
| project TimeGenerated, Computer, Account, Process
```

---

### 10. Lateral Movement Detection

Detect remote logons across systems.

```kql
SecurityEvent
| where EventID == 4624
| where LogonType == 3
| summarize count() by Account, Computer, IpAddress
| sort by count_ desc
```

---

### 11. RDP Logon Activity

Monitor Remote Desktop usage.

```kql
SecurityEvent
| where EventID == 4624
| where LogonType == 10
| project TimeGenerated, Account, Computer, IpAddress
```

---

### 12. Multiple Systems Accessed by a Single Account

Detect potential lateral movement.

```kql
SecurityEvent
| where EventID == 4624
| summarize SystemsAccessed=dcount(Computer) by Account
| where SystemsAccessed > 10
```

### 13. External Sign-In Activity (Microsoft Entra ID)

Identify sign-ins from unusual locations.

```kql
SigninLogs
| summarize SignInCount=count() by UserPrincipalName, IPAddress, Location
| sort by SignInCount desc
```

## 14. Multiple Failed Cloud Sign-Ins

Detect password-spraying attacks against Entra ID accounts.

```kql
SigninLogs
| where ResultType != 0
| summarize FailedAttempts=count() by UserPrincipalName, IPAddress
| where FailedAttempts > 10
```

---

### 15. Impossible Travel Detection

Identify user sign-ins from geographically distant locations within a short period.

```kql
SigninLogs
| sort by UserPrincipalName, TimeGenerated asc
| project TimeGenerated, UserPrincipalName, Location, IPAddress
```

> Typically enhanced with geolocation calculations and UEBA analytics.

## Recommended Threat Hunting Queries

For most SOC investigations, start with:

1. Failed Logon Attempts (4625)
2. Successful Logon After Failures
3. Privileged Group Membership Changes
4. New User Creation
5. PowerShell Execution
6. Encoded PowerShell Commands
7. Suspicious Command-Line Activity
8. RDP Logons
9. Lateral Movement Detection
10. Multiple Failed Entra ID Sign-Ins
11. Microsoft Defender Alerts
12. CrowdStrike Detection Events
13. Rare Process Execution
14. Time-Series Event Analysis

These provide the highest value for detecting brute force attacks, privilege escalation, lateral movement, malware activity, credential abuse, and endpoint compromise.

### Find powershell execution events that could involve download

`
Union DeviceProcessEvents, DeviceNetworkEvents
| where Timestamp \> ago(7d)
| where FileName in~ (“powershell.exe”, ”powershell_ise.exe”)
| where ProcessCommandLine has_any(“WebClient”, "DownloadFile", "DownloadData", "DownloadString", "WebRequest", "Shellcode", "http", "https")
| project Timestamp, DeviceName, InitiatingProcessFileName, InitiatingProcessCommandLine, FileName, ProcessCommandLine, RemoteIP, RemoteUrl, RemotePort, RemoteIPType
| top 100 by Timestamp
`

### Credential Access/Private Key Files

This query identifies file operation with files having one of the extensions commonly used to save a private key. The risk is that if an attacker were to obtain the file, they could brute force any password on it and
potentially obtain a powerful certificate. To do this, they would only need to obtain read access to the file. The risk associated with these files is heavily determined by the value of the certificate. For example, loss of a self-signed certificate created by Adobe Acrobat is significantly less impactful than loss of a website SSL certificate. Recommendation: Know where these files are, and if possible, back them
up and remove them. You might also consider creating an informational alert

DeviceFileEvents
| where Timestamp \> ago(7d)
| where FileName endswith '.pfx' or FileName endswith '.pfn' or FileName endswith '.p12'

### Number of security events logged by device type, operating system (Windows, linux), and severity level

The exact KQL depends on your data source. For Microsoft Sentinel, the most common approaches are:

#### Option 1: Using `SecurityAlert` (Recommended for Severity-Based Security Events)

```kql
SecurityAlert
| extend DeviceType = iff(isempty(CompromisedEntity), "Unknown", CompromisedEntity)
| summarize EventCount = count()
    by DeviceType,
       OSPlatform,
       AlertSeverity
| order by EventCount desc
```

---

#### Option 2: Using `SecurityEvent` (Windows Security Logs)

`SecurityEvent` does not contain a native severity field, so you must derive one based on Event IDs or your organization's classification.

```kql
SecurityEvent
| extend OS = "Windows"
| extend Severity =
    case(
        EventID in (4625, 4720, 4728, 4732, 4756), "High",
        EventID in (4624, 4634, 4648), "Medium",
        "Low"
    )
| summarize EventCount = count()
    by Computer,
       OS,
       Severity
| order by EventCount desc
```

---

#### Option 3: Using Microsoft Defender XDR (`DeviceEvents`)

This is typically the best option when you want device type, operating system, and severity together.

```kql
DeviceEvents
| extend Severity =
    case(
        ActionType has_any ("MalwareDetected", "AntivirusDetection"), "High",
        ActionType has_any ("Suspicious", "Exploit"), "Medium",
        "Low"
    )
| summarize EventCount = count()
    by DeviceType,
       OSPlatform,
       Severity
| order by EventCount desc
```

---

#### Option 4: Unified Dashboard Query (Windows & Linux Devices)

If your environment is onboarded to Microsoft Defender for Endpoint:

```kql
DeviceInfo
| summarize arg_max(Timestamp, *) by DeviceId
| project DeviceId, DeviceType, OSPlatform
| join kind=inner (
    DeviceEvents
    | summarize EventCount = count() by DeviceId, ActionType
) on DeviceId
| extend Severity =
    case(
        ActionType contains "Malware", "High",
        ActionType contains "Exploit", "High",
        ActionType contains "Suspicious", "Medium",
        "Low"
    )
| summarize TotalEvents = sum(EventCount)
    by DeviceType,
       OSPlatform,
       Severity
| order by TotalEvents desc
```

#### Visualization for Sentinel Workbook

To create a chart showing security events by device type, OS, and severity:

```kql
DeviceInfo
| summarize arg_max(Timestamp, *) by DeviceId
| project DeviceId, DeviceType, OSPlatform
| join kind=inner (
    DeviceEvents
    | summarize EventCount = count() by DeviceId
) on DeviceId
| summarize TotalEvents = sum(EventCount)
    by DeviceType,
       OSPlatform
| render columnchart
```

This will produce a dashboard-friendly view showing:

* **Device Type** (Workstation, Server, Mobile, etc.)
* **Operating System** (Windows, Linux, macOS)
* **Severity Level** (High, Medium, Low)
* **Number of Security Events** per category.
