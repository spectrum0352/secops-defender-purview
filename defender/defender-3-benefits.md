
# How Microsoft Defender detects threats?

- **Data Collection & Correlation:** Defender solutions gather data from various sources (endpoints, networks, cloud) and correlate it to identify threat patterns and relationships. For example, linking endpoint activity with network traffic to detect communication with malicious IPs.
- **Statistical Anomaly Detection:** Defender uses statistical analysis to identify deviations from normal behavior, such as sudden network traffic spikes or unusual user account activity, which may indicate a threat.
- **Threat Intelligence Integration:** Defender incorporates threat intelligence feeds to identify known threats and their indicators of compromise (IOCs), like new malware variants or phishing campaigns.
- **Machine Learning for Threat Identification:** Defender employs machine learning models, trained on historical security data (both malicious and benign), to accurately detect new, unseen threats. For example, a model trained to identify malware samples can scan new files for malicious code, even if the files are previously unknown.
- **Automated Investigation & Response:** Machine learning in defender can automate threat investigation, helping determine root cause, and suggesting remediation actions, speeding up response times.
- **Enhanced Visibility:** By collecting data from diverse sources, defender provides a comprehensive view of the security landscape, revealing threats that traditional solutions might miss.
- **Reduced False Positives:** Analytics and machine learning help minimize false positives, allowing security teams to focus on genuine threats and reducing wasted time.
- **Accelerated Detection & Response:** Automation through defender enables faster threat detection and response, improving overall security effectiveness.
